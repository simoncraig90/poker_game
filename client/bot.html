Poker Bot — Browser Automation

Each bot runs in its own Chrome window with a real mouse cursor. It clicks buttons, types bet amounts, and reads cards from the screen — exactly like a human player.

Quick Start

# Start the poker server first
node src/server/ws-server.js

# In another terminal, launch 3 bots
node src/bot/browser-bot.js --bots=3

Three Chrome windows will open, each navigating to the table. You'll see cursors moving to buttons and clicking them in real time.

Options

--url=http://localhost:9100   # Server address
--bots=3                     # Number of bots (2-6)
--style=TAG                  # TAG, LAG, ROCK, FISH, or MIXED
--delay=1500                 # Think time in ms
--hands=50                   # Stop after N hands (0 = unlimited)
--headless                   # No visible browser
--chrome="/path/to/chrome"   # Chrome executable path

Examples

# 4 bots with mixed styles, slower for watching
node src/bot/browser-bot.js --bots=4 --style=MIXED --delay=2000

# 2 tight bots, 20 hands only
node src/bot/browser-bot.js --bots=2 --style=TAG --hands=20

# Specify Chrome path on Windows
node src/bot/browser-bot.js --chrome="C:\Program Files\Google\Chrome\Application\chrome.exe"

Play Styles

TAG   — Tight-Aggressive (solid, selective, bets hard)
LAG   — Loose-Aggressive (wider range, more bluffs)
ROCK  — Ultra-tight (only premium hands)
FISH  — Loose-passive (calls too much)
MIXED — Each bot gets a different style
How it works: Each bot is a separate Chrome process controlled by Puppeteer. It moves the mouse to buttons (Fold, Call, Raise), clicks at the pixel coordinates, and types bet amounts character by character. The server cannot distinguish a bot from a human — the input path is identical.
Note: Showdown is not yet implemented in the engine (GAP-1). Hands where 2+ players reach the river will error. This is an engine limitation, not a bot issue.
